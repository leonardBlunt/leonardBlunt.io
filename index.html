Funds Transfer Cloud Native Integration

Use Case: Funds Transfer Scenario

The bank needs to support the customers need to transfer payments/funds to registered receivers when the transfer is received at the backend the customer needs to be notified that the transfer was initiated. Then once the processing is complete the customer will receive a second notification that indicating the processing outcome

Funds Transfer Scenario - Sequence of Events

  1. User triggers a transfer of funds from their mobile banking
  2. Backend for Frontend service invokes call to backend systems invokes the API
  3. API invokes persistent queue on the queue manager
  4. The queue manager is configured to work with streaming queue which will post the message to the events for processing by the subscribed event consumers
  5. Events are processed on the event brokers
  6. Event consumers process the inbound event
  7. Core banking processes the transfer
  8. When processing is complete core banking triggers notification to the events to be picked up by the event consumers
