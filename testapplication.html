Application testing

Testing the funds transfer scenario involves validating the end-to-end flow of events from user initiation to the completion of the transfer process in a banking application. The testing approach should ensure the functionality, performance, reliability, and security of the entire funds transfer system.

  1. MQ Messaging Request and Reply

    Testing an MQ-JMS (Message Queue - Java Message Service) client application is crucial to ensure the reliability, performance, and correctness of the messaging system. MQ-JMS is widely used for building distributed and decoupled applications that communicate through message queues. This overview outlines key aspects and considerations for testing a MQ-JMS client application.

We will now test the mq messaging through JMS application.

  1. Clone the following repositry from github.
    2. wget https://github.com/sujatha-sureshkmr/ft-mq-jms/archive/refs/heads/main.zip
    unzip main.zip
    cd ft-mq-jms-main/
  2. Setup environment variables. Note: You may change the currency, rate and the amount. Below is just a sample.
    3. export qmhostname=$(oc get route -n cp4i fundtransfer-ibm-mq-qm -o jsonpath="{.spec.host}")
                                            
     export QUEUE_NAME=TRANSFER.REQUEST
                                            
     export FromCurr=SGD
                                            
     export ToCurr=USD
                                            
     export CurrencyRate=0.74
                                            
     export TransferAmount=100
  3. We will now execute jms application. Note: ensure the ftpapp.jks and trust.jks files are available in the parent folder
    4. java -Djavax.net.ssl.keyStoreType=jks -Djavax.net.ssl.keyStore=ftappkey.jks -Djavax.net.ssl.keyStorePassword=password      -Djavax.net.ssl.trustStoreType=jks -Djavax.net.ssl.trustStore=trust.jks -Djavax.net.ssl.trustStorePassword=password      -Dcom.ibm.mq.cfg.useIBMCipherMappings=false      -cp ./lib/com.ibm.mq.allclient-9.2.4.0.jar:./lib/javax.jms-api-2.0.1.jar:./lib/json-20211205.jar:.      com.ibm.mq.samples.jms.JmsPut
  4. Now, we will login to MQ console to verify if the messages are requested and sent.
    Go to cp4i console -> Integration Instances -> fundtransfer -> Manage -> fundtransfer (queue manager)
  5. You will see messages are bring sent over and received replies.
    Note: TRANSFER.REQUEST refers to the message initiated through user action.
    TRQ.COPY refers to the message copied from TRANSFER.REQUEST.
    TRANSFER.REPLY refers to the message received from the application upon successful fund transfer.
  6. Let us look at the TRANSFER.REQUEST MESAGES. Click "TRANSFER.REQUEST" queue -> timestamp. You will see the MQ message that is sent upon fund transfer initiation. You may also click "Download" to export the message in a text file.
  7. Let us look at the TRQ.COPY MESAGES. Click "TRQ.COPY" queue -> timestamp. You will see the MQ message that is copied upon fund transfer initiation from "TRANSFER.REQUEST". You may also click "Download" to export the message in a text file.
  8. Let us look at the TRANSFER.REPLY MESAGES. Click "TRANSFER.REPLY" queue -> timestamp. You will see the MQ message that is sent upon successful completion of fund transfer. You may also click "Download" to export the message in a text file.
