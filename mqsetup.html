Configure Messaging Queue

  1. Overview

A Native HA configuration provides a highly available queue manager where the recoverable MQ data (for example, the messages) are replicated across multiple sets of storage, preventing loss from storage failures. The queue manager consists of multiple running instances, one is the leader, the others are ready to quickly take over in the event of a failure, maximizing access to the queue manager and its messages.

We will now setup and configure MQ queue manager and queues.


  1. ConfigMap

The ConfigMap API object holds key-value pairs of configuration data that can be consumed in pods or used to store configuration data for system components such as controllers. ConfigMap is similar to secrets, but designed to more conveniently support working with strings that do not contain sensitive information. Let us configure the config map.

  1. Go to ocp console -> Workloads -> ConfigMaps
  2. We are working on the project “cp4i”. Click create ConfigMap
  3. Use the following options for the setup and click “Create”
    4. ConfigMap Options
    Option Value
    Configure via Form view
    Name nativehamqsc
    Key fundtransfer.ini
    Value Service: Name=AuthorizationService
    EntryPoints=14
    SecurityPolicy=UserExternal
    Key nativehamqsc.mqsc
    Value define ql(TRANSFER.REQUEST) REPLACE DEFPSIST(YES)


    define ql(TRANSFER.REPLY) REPLACE DEFPSIST(YES)


    DEFINE CHANNEL(FUNDTRANSFERCHL) CHLTYPE(SVRCONN) TRPTYPE(TCP) SSLCAUTH(REQUIRED) SSLCIPH('ANY_TLS12_OR_HIGHER')

    ALTER AUTHINFO(SYSTEM.DEFAULT.AUTHINFO.IDPWOS) AUTHTYPE(IDPWOS) CHCKCLNT(OPTIONAL)

    set chlauth(FUNDTRANSFERCHL) TYPE(SSLPEERMAP) SSLPEER('CN=fundtransfer') USERSRC(MAP) MCAUSER('fundtransfer') ACTION(ADD)

    SET AUTHREC PRINCIPAL('fundtransfer') OBJTYPE(QMGR) AUTHADD(CONNECT,INQ)

    SET AUTHREC PROFILE('transfer.request') PRINCIPAL('fundtransfer') OBJTYPE(QUEUE) AUTHADD(BROWSE,GET,INQ,PUT)

    SET AUTHREC PROFILE('transfer.reply') PRINCIPAL('fundtransfer') OBJTYPE(QUEUE) AUTHADD(BROWSE,GET,INQ,PUT)

    DEFINE ql(TRQ.COPY) REPLACE

    ALTER ql(TRANSFER.REQUEST) STREAMQ(TRQ.COPY)

    DEFINE CHANNEL(KAFKA.SVRCONN) CHLTYPE(SVRCONN) TRPTYPE(TCP) SSLCAUTH(OPTIONAL) SSLCIPH('ANY_TLS12_OR_HIGHER') REPLACE

    SET CHLAUTH(KAFKA.SVRCONN) TYPE(BLOCKUSER) USERLIST('event-admin') WARN(YES) ACTION(REPLACE)

    REFRESH SECURITY

    SET AUTHREC OBJTYPE(QMGR) GROUP('kafkausers') AUTHADD(CONNECT, INQ)

    SET AUTHREC OBJTYPE(QUEUE) PROFILE(TRQ.COPY) GROUP('kafkausers') AUTHADD(ALLMQI)
  4. ConfigMap is successfully created in namespace “cp4i”

    1. Queue Manager

    A queue manager is a program that provides messaging services to applications. Applications that use the Message Queue Interface (MQI) can put messages on queues and get messages from queues. The queue manager ensures that messages are sent to the correct queue or are routed to another queue manager. Let us now login to CP4I navigator console to configure queue manager

    1. Go to the environment details page -> “Cloud Pak Console’ -> copy and paste “login info for the CP4I Navigator console” in a text editor. You will need this information throughout this lab.
    2. Login to CP4I Navigator console. Log in with “IBM provided credentials (admin only)”. Use the credentials given in the environment details
    3. From the welcome screen -> click Integration Instances
    4. Click “Create Instance”
    5. Click “Messaging” -> Next
    6. Click “Production (Custom)” -> Next
    7. Follow the following configurations:
      1. UI Form is chosen by default. Enable advanced settings
      2. Name the queue manager as “fundtransfer”. Follow the options highlighted in red for licence configuration
      3. PKI configuration – follow the highlighted configurations
      4. Trust key configuration
      5. Choose Native HA for queue manager availability
      6. Settings for supplying INI for the Queue Manager.
      7. Follow the red highlighted parts for config map configuration
      8. Storage class configuration
      9. Storage for queue manager and recovery logs
      10. Configuration for security context
      11. Configuration for version and web
      12. Now click create for queue manager creation.
      13. It would take 5 to 10 mins for the queue manager to be active
